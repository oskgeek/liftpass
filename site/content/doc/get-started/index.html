Get Started

You should be able to build Liftpass on most Linux-based systems.

Get the build dependencies

Liftpass is developed in Python 3.x. There are a number of modules required for running the server, however all are available through pip and easy_install. In a fresh Ubuntu installation run:

{% highlight 'bash' %} sudo apt-get install git python3-setuptools build-essential python3-dev libmysqlclient-dev python3-pip nodejs {% endhighlight %}

Get the source code

Download or clone the Liftpass repository: {% highlight 'bash' %} # Clone the repository git clone https://github.com/whiteshoemedia/gondola-core.git # Download repository as a zip file wget https://github.com/whiteshoemedia/gondola-core/archive/master.zip unzip master.zip {% endhighlight %}

Install dependencies

For most Linux-based systems pip can automatically install all required Python module dependencies. {% highlight 'bash' %} cd liftpass sudo pip install -r requirements.txt {% endhighlight %}

Configuring Liftpass

Liftpass is configured via the config.py file. This file is generated by running the configure command. At this time default required directories will also be created.

{% highlight 'bash' %} python manage.py configure {% endhighlight %}

Database

Once the config.py file has been generated you can build Liftpass's database. This will create all required tables that Liftpass uses.

{% highlight 'bash' %} python manage.py content create {% endhighlight %}
By default Liftpass is configured to run locally. As such, it uses local storage and SQLite for database which will not scale or manage with higher traffic loads. More information on configuring Liftpass for deployment can be found at Liftpass Server documentation page.

Starting the server

You are now ready to manually start running ths server.

{% highlight 'bash' %} python manage.py API start {% endhighlight %}

Testing

To make sure everything has been installed and configured correctly it is encouraged that you test Liftpass.

{% highlight 'bash' %} python manage.py test all {% endhighlight %}

All unit-tests should complete successfully and without errors. Because the tests will not cleanup after themselves it is encouraged that you re-create the Liftpass database:

{% highlight 'bash' %} python manage.py content flush python manage.py content create {% endhighlight %} {% endblock %}
